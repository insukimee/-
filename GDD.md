# 게임 기획서 (GDD) - 프로젝트명 미정

> 워킹 타이틀: "Ragdoll Party" (임시)
> 장르: 2D 물리 기반 파티 격투 게임 (갱비스트 스타일)
> 인원: 2~10인 온라인 멀티플레이
> 엔진: Godot 4.x

---

## 1. 한 줄 컨셉
흐물흐물한 랙돌 캐릭터로 서로 밀치고 넘어뜨려 스테이지 밖으로 떨어뜨리거나
목표를 달성하는 2D 파티 격투 게임.

## 2. 핵심 재미 (Core Fun)
- 정교한 조작이 아니라 "의도대로 안 되는" 물리 반응에서 나오는 웃음
- 예측 불가능한 넘어짐, 밀림, 충돌
- 친구들과 같이 봐도 재밌는 관전성 (스트리밍 친화적)

## 3. 조작
| 키 | 동작 |
|---|---|
| 방향키/WASD | 이동 |
| Space | 점프 |
| Shift | 잡기/붙잡기 (양손 각각 가능하면 좋음, 1차는 단일 그랩) |
| 마우스 좌클릭 (또는 별도 키) | 밀기/펀치 |

- 1차 버전은 "이동 + 점프 + 잡기 + 밀기" 4개 액션으로 최소화
- 콤보 시스템 없음. 복잡한 커맨드 지양

## 4. 게임 모드 (우선순위 순)
1. **낙사전 (King of the Hill 변형)** - 좁은 스테이지에서 서로 밀어 떨어뜨리기, 마지막 생존자 승리
2. **점령전** - 특정 구역 오래 버티기
3. **협동 미니게임** (2차 개발) - 같이 물건 옮기기 등

## 5. 스테이지 기믹 (우선순위)
1. 좁아지는 발판 (시간 지나면 바닥이 줄어듦)
2. 컨베이어 벨트
3. 회전 장애물
4. 낙사 존 (스테이지 하단 = 즉시 탈락)

## 6. 캐릭터 설계
- RigidBody2D 기반 관절 구조: 몸통(Torso) - 머리(Head) - 팔(Arm x2) - 다리(Leg x2)
- PinJoint2D / DampedSpringJoint2D로 관절 연결
- 색상 커스터마이징만 (스킨 시스템은 2차)

## 7. 멀티플레이 구조
- **호스트 권위(Host Authoritative)** 방식
- 물리 연산은 호스트(방장)만 계산, 나머지 클라이언트는 위치/회전 값만 수신
- Godot MultiplayerSynchronizer + ENetMultiplayerPeer 사용
- 최대 10인 룸, 티어별 서버 없이 P2P 호스트 방식으로 1차 구현 (규모 커지면 전용 서버 검토)

## 8. 개발 마일스톤
- [ ] M1: 싱글 캐릭터 물리 프로토타입 (로컬)
- [ ] M2: 로컬 2인 (같은 PC, 다른 컨트롤러/키)
- [ ] M3: 스테이지 1개 + 기믹 1개
- [ ] M4: 네트워크 멀티플레이 (2인)
- [ ] M5: 네트워크 멀티플레이 (최대 10인 확장)
- [ ] M6: UI/로비/매칭
- [ ] M7: 사운드/이펙트
- [ ] M8: 밸런스 및 버그 수정
- [ ] M9: 빌드 및 배포 (Steam or itch.io)

## 9. 리스크 및 주의사항
- 물리 기반 게임의 네트워크 동기화는 난이도가 높음 → 호스트 권위 방식 고수, 클라이언트 사이드 물리 예측은 후순위
- 랙돌 관절이 너무 많으면 성능 저하 → 관절 수는 5개 내외로 제한
- 바이브코딩 시 물리/네트워크 버그는 원인 추적이 어려우므로 기능 단위로 자주 커밋하고 테스트할 것
